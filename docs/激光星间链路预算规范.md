> 版本：v1（2026-07）　作者：评估+多源调研综合
> 目的：给再生式链路预算模块的**星间激光**体制一份**第一性原理、逐条有据可循、无模糊边界**的规范，

## 0. 结论摘要（TL;DR）

- 现有激光模块 **不是"全错"**：dB 功率链的**结构**与 MathWorks 官方光学 ISL 示例互证无误（`G=20lg(πD/λ)`、`L_fs=20lg(4πd/λ)`）。
- **真正的错在三层**：① 信号/检测层——注释否认载波、`N₀` 取 `hν`（理想量子极限）而非 `hν·F`、调制与灵敏度完全脱钩；② 灵敏度/门限——直检与相干两套口径被混用；③ **星间几何对"激光"实质不完整**——只有径向量、缺提前量角/跟踪转率、最差量取自单点。
- **体制拍板（已定）**：默认主体制 = **直检 IM/DD（对标 SDA-OCT / Mynaric CONDOR 真实在轨交叉链）**；相干 DP-QPSK 100G 降为**明确标注的"下一代/实验室"备选预设**。
- **GitHub 调研结论**：不存在可直接移植的"相干激光 ISL 预算"开源实现；功率链/几何可照 MathWorks + arXiv + DLR 逐条复刻，灵敏度/几何补全须照 IEEE/CCSDS/DLR 文献自研。

**最高优先级的三条硬修正**：
1. 直检 OOK 的灵敏度 ≈ **220–250 光子/bit**，绝不是相干的 20；调制下拉必须驱动灵敏度（现在选 OOK 仍算 20 是 critical bug）。
2. 相干口径下 `N₀ = hν·F`（F≈3.16，EDFA 前放 NF≈5dB），现用 `N₀=hν` 至少低估 3 dB。
3. 几何必须在**惯性系**输出横向相对速度 `v⊥` → 提前量角 `θ_PA=2v⊥/c`（LEO≈50µrad，与光束同量级）、跟踪转率 `ω=v⊥/range`；各最差量**独立取各自极值**（现取"最大 range 单点"，恰是径向速率转向点 → 多普勒≈0）。

## 1. 体制界定（最重要的边界，不留模糊）

**检测体制决定噪声口径与灵敏度来源，两者不可共用一个灵敏度数字。** 真实工程分三档：

| 体制 | 代表终端 | 检测 | 波长 | 速率 | 灵敏度/噪声口径 |
|---|---|---|---|---|---|
| **直检 IM/DD**（★主预设） | SDA-OCT / Mynaric CONDOR（美军 T1/T2 大网） | 直接检测（光电二极管，无本振） | 1550nm（C 波段 DWDM） | 2.5–10 Gbps | **测量 P_req** 或辐照度门限；不显式构造 N₀ |
| 相干 homodyne | Tesat（EDRS/SpaceDataHighway heritage） | 相干零差 BPSK（本振激光） | 1064nm | 1.8–5.6 Gbps | `N₀=hν·F`，所需 Eb/N₀ |
| 相干 DP-QPSK 100G | Nature/TBIRD 实验室演示 | 相干零差 DP-QPSK | 1550nm | 100G（27.95 Gbaud） | `N₀=hν·F`，门限≈−30dBm |

**关于"有没有载波/信号概念"**（用户原始疑问的权威回答）：
- **直检 OOK/PPM**：把信息编码在**光强**（有光/无光），接收端只测功率，**近似"能量/光子受限、无载波"**——这时"无载波"说法才成立。
- **相干 BPSK/DP-QPSK**：把信息编码在**光载波的相位**（光载波 = 激光电场，ν=c/λ≈**193.4 THz**），接收端用**本振激光零差/外差拍频 + 载波相位恢复（OPLL/DSP）**，与 RF 相干接收机**数学完全同构**——**恰恰是把"载波"用到极致的体制**。电域带宽 ≈ 符号率。
- 所以现有代码注释"激光无载波、非带宽受限"**对默认的相干 DP-QPSK 是概念错误**；引擎自己按 `Δf=v_r/λ` 算 193THz 载波多普勒并告警，本身就以"存在载波"为前提。

> **规则**：模块必须显式携带"检测体制"字段（direct / coherent-homodyne / coherent-heterodyne）；该字段驱动 §2 的灵敏度分支，严禁跨体制套用。

## 2. 功率预算（直检/相干共用几何与光学，dB 加性链）

```
P_rx[dBm] = P_tx + G_tx + G_rx − L_fs − L_opt,tx − L_opt,rx − L_point
（星间真空段：无大气/雨/云/闪烁——两端在真空、LOS 不穿对流层，此简化正确）
```
> 结构与 MathWorks 官方 `Prx=Ptx+OEtx+OErx+Gtx+Grx−LPtx−LPrx−LPS` 一致 ✔。逐项：

### 2.1 接收孔径增益 `G_rx = 20·lg(π·D_rx/λ)`
- **正确，保留。** MathWorks `grx=(πd/λ)²`、DLR eq18、arXiv `G_R=(πD_R/λ)²` 三源互证。
- 例：D=80mm、λ=1550nm → **104.2 dBi**（本仓库数值已核对无误）。

### 2.2 发射增益 `G_tx` —— ⚠️ 需改
- **问题**：现用 `20·lg(π·D_tx/λ)`（衍射极限满口径峰值），却又独立给"全发散角 47µrad"——**自相矛盾**：一旦为指向鲁棒把光束展宽，孔径式会高估 TX 增益。
- **正确做法（由发散角定义）**：`G_tx = 16/Θ_T²`（Θ_T = 全发散角，rad）[arXiv 2204.13177]；等价 DLR eq10 `g_Tx = 10·lg((4√ln2/θ_FWHM)²)`。
- 孔径式仅作**衍射上界校验**：`Θ_T ≥ ~4λ/(πD)` 时两式一致；实际 Θ_T 更大 → 用发散角式。

### 2.3 自由空间损耗 `L_fs = 20·lg(4π·d/λ)`
- **正确，保留。** MathWorks / DLR eq13 / arXiv 三源互证。与两端 `(πD/λ)²` 配对后净为物理捕获比 `(D_tx·D_rx/(λd))²·(π²/16)`。
- ⚠️ 现引擎注释写"净为 `(D_tx·D_rx/(λd))²`"**漏了 π²/16 = −2.09 dB**——数值无误、注释误导，须补系数或改写为 Friis 光学形式。

### 2.4 光学效率 `L_opt = −10·lg(η)`
- **与增益解耦作独立 dB 项，正确。** MathWorks `OE=10lg(η)`，缺省 η=0.8→**−0.97 dB/端**；本仓库用 2.5 dB/端更保守（含耦合）。
- **相干额外项**：单模光纤/本振混频耦合损耗须单列（Walsh 实测耦合 −8 dB 作上界，典型 −1~−3 dB），不要笼统塞进 2.5 dB。

### 2.5 指向损耗 `L_point` —— ⚠️ 需改（低估约 4×）
- **问题**：现用 `10·lg(1+4(σ/θ_e)²)`，`θ_e=θ_div/2`；而 `θ_div=47µrad=2.44λ/D` 是**艾里首零级全角**，`θ_div/2=1.22λ/D` 约为真实高斯 1/e² 半角的 **2 倍** → 因损耗 ∝ 1/θ_e²，**指向抖动损耗与衰落被低估约 4×**；同一错误 θ_e 还进了可用度系数 34.74。
- 校验：MathWorks 静态指向式 `L_pt[dB]=4.3429·g·θ²`（g=线性孔径增益，θ=rad）给 80mm/1550/2µrad → **0.46 dB**，而现引擎给 0.12 dB，差 ~4×。
- **正确做法（二选一，须统一 σ 定义=每轴 rms）**：
  - 瞬时确定性：高斯指数式 `L = exp(−16·θ²/Θ_div²)`（dB 化）[arXiv]；
  - 抖动均值（更严谨）：DLR β-分布均值式 `a_BW = 10·lg(β/(β+1))`，`β = θ_FWHM²/(4ln2·σ²)`。
- **确定性偏置**（安装残差 + 提前量残余，见 §3）另作一项叠加，不并入抖动均值。

## 3. 灵敏度 / 门限（分体制，严禁混用）

### 3.1 直检 IM/DD（★主预设）
- 灵敏度 = **测量/曲线拟合的所需接收光功率 P_req(dBm)** 或**所需每比特光子数 ppb**；热/散粒/背景噪声已经验地折进该单点，**不显式构造 N₀**。
- 权威锚：
  - MathWorks OOK：**P_req = −35.5 dBm @ 10 Gbps, BER 1e-12**（反算 ≈ **220 光子/bit**）。
  - Giggenbach（DLR）：**250 光子/bit @ BER 1e-3 + RS(255,223)**，`P_1E-3 = 250·R·hc/λ`。
  - SDA-OCT v3.0.1：以**接收孔径辐照度门限 ≥ 25 µW/m² @ 5500km**（OCT-037）表述，非 ppb。
- ⚠️ 现引擎默认 PPB=20 是**相干**值，用在直检会虚高灵敏度 ~10 dB。

### 3.2 相干（备选预设）
- **噪声底 `N₀ = hν·F`**，F = 2·n_sp = 光放大器/相干前端噪声因子：量子地板 **F=2（3 dB）**[Caves 1982]；C 波段 EDFA 前放实测 **NF≈5 dB → F≈3.16**[JLT 32(7):1295]。现用 `N₀=hν` **至少低估 3 dB**。
- 灵敏度 = 量子极限 + 实现损耗 + F：
  - 量子极限（理想零差 shot-noise）：**QPSK/BPSK ≈ 9 光子/bit @ BER 1e-9**（per-bit；per-symbol=18，**须区分**）；外差 = 18；零差比外差好 3 dB（判据 `BER=½erfc(Q/√2)`，Q≈6@1e-9）。DP-QPSK 每偏振 ≈ 单偏振 QPSK。
  - 工程灵敏度 ≈ 20–40 光子/bit（现默认 20 落区间下沿，合理但偏乐观；⚠️ 无厂商公开值，纯物理推导，是余量最大不确定项 ±1dB/25%）。
  - 真实 100G 锚：Walsh（Nature）门限 ≈ **−30 dBm @ FEC-BER 4.5e-3**，27.95 Gbaud。
- **正确所需功率式**：`P_req[dBm] = 10·lg(ppb_eff·hc/λ·R) + 30`，其中 `ppb_eff` = 量子极限 × F × 实现损耗；**更严谨路径**是显式 `N₀=hν·F → 所需 Eb/N₀（QPSK+实现损耗）→ P_req`，让灵敏度与调制/FEC 挂钩。
- ⚠️ 现 `C/N₀=P_rx−10lg(hν)`、`Eb/N₀=C/N₀−10lg(R)` 报出的是**"收到的每比特光子数"（光子极限参考）**，不是真噪声参考的 Eb/N₀——**要么重标签为"光子极限参考(N₀=hν)"，要么改用 hν·F**。

### 3.3 调制 → 灵敏度映射表（消除"调制脱钩"bug）
| 调制 | 检测 | bit/symbol | 典型所需光子/bit @1e-9（FEC 前工作点更低） |
|---|---|---|---|
| OOK-NRZ | 直检（PIN/APD） | 1 | ~220–1000（预放大 ~40） |
| DPSK | 直检/预放大 | 1 | ~20–40 |
| BPSK | 相干零差 | 1 | 9（量子极限）→ 工程 20–40 |
| QPSK | 相干零差 | 2 | 9/bit（量子极限）→ 工程 20–40 |
| **DP-QPSK** | 相干零差 | 4 | ≈单偏振 QPSK/bit；符号率=R/4 |

> 选定调制即写入对应默认灵敏度 + 符号率 + 是否走相干（LO/线宽约束），允许高级覆盖。**杜绝"选 OOK 仍算 20 ppb"**。

## 4. 星间几何完整量清单（本仓库最大缺口，用户点名）

现 `solveIslWorstCase` 只输出**径向量**（range / 径向 range-rate / 地心夹角 / 掠地高度 / 时延 / 互视占比），且所有派生量取自"**最大 range 单点**"——而最大 range 点正是径向距离变化率**转向点**（dρ/dt≈0），使多普勒近零（bug 已确认）。

**正确应输出，且各"最差量"独立取各自极值（不能同点取值）：**

### 4.1 帧前提（修复的正确性基石）
- range 是帧不变量，故现有径向多普勒无误；但 `v⊥ / θ_PA / ω` 都是**帧相关量**，**必须在惯性系（ECI/TEME）对两星速度矢量做差**（`satellite.js` 的 `pv.velocity` 本就是 TEME 惯性速度）。
- **绝不能在 ECEF 里做速度差分**，否则注入地球自转 `ω⊕×r`（近地 ~0.46 km/s）的虚拟横向速度，污染 PAA/ω。

### 4.2 应输出的量
| 量 | 公式 | 取值时刻 | 用途 / 依据 |
|---|---|---|---|
| 最差 FSL | max(range) | range 极大刻 | 功率预算 |
| 最差多普勒 | `Δf=v_r/λ`，`v_r=v_rel·û` | **max\|v_r\| 独立极值刻** | 与 `dopplerLockGHz` 比对 [SDA OCT-023 要求全程建模] |
| 横向相对速度 | `v⊥=\|v_rel − v_r·û\|` | max(v⊥) 刻 | 驱动下面两量 |
| **提前量角** | `θ_PA = 2·v⊥/c`；Az/El 分解 `β_A=2r₁·Ȧ·cosE₁/c`、`β_E=2r₁·Ė/c` | max 刻 | LEO≈50µrad（500km LEO≈51、GEO≈18、LEO-GEO 66–72），**与光束发散 47µrad 同量级**，光学 ISL 独有；未补偿=确定性指向损耗 [MDPI 12(9):886, DLR] |
| 跟踪转率 | `ω = v⊥/range` | max 刻 | FSM 带宽/万向节转率需求（DLR 天顶可超 ~1°/s） |
| 太阳角 | ∠(收/发 boresight, 太阳矢量) 时间序列 | 全程 | 太阳规避（SDA 参考几何：日月不在 boresight 30° 内） |

## 5. 可用度正确合成（不留模糊）

```
完整可用度 A_sys = 几何互视占比 × 指向抖动可用度 × 太阳规避可用度
建链/重捕获 = 独立"中断"报告项（锁定即进跟踪，不并入稳态）
```
- **几何互视**：LOS 未被"地球 + 大气余量"遮挡的时间占比（现已有）。注：`atmMargin=100km` 对纯真空光 ISL 偏保守，建议按光/微波分设默认（光 ~30–40km，掠地切线落入对流层才触发吸收/闪烁）。
- **指向抖动衰落**：Rayleigh 径向抖动 → 均值损耗（进预算）+ 中断概率（进可用度）。严谨式 `P_out=(h_th/A₀)^(γ²)`，`γ=w_zeq/(2σ)` [Farid-Hranilovic JLT 2007]；现用 `θ_th=θ_div√(margin/34.74)` + `P_fade=exp(−θ_th²/2σ²)` 是同族简化，可保留但须注明近似，并修正 34.74 里的 θ_e 定义（见 §2.5）。
- **太阳规避**：太阳落入接收视场半锥角（默认 20°）内的时间占比须扣除——现仅报告不并入，"完整"口径应至少可选并入（须真实日月星历，避免与建链/抖动重复计中断）。
- **建链/重捕获**：≤ acqTime @ 粗锥 @ 成功率，独立报告（现做法正确）。
- **硬门限（工程锚）**：链路余量 **≥ 3 dB**，且**通信 + PAT 双通道皆需** [SDA-OCT OCT-030/031]；`designMargin` 现声称合格判定却不 gate → 合格判定改 `ok=(margin≥designMargin)`。BER 目标经 FEC 后定义并绑定码率 → 有效速率。

## 8. 待拍板的开放问题

1. **直检灵敏度基准**：MathWorks −35.5dBm 是 10Gbps OOK 值，SDA 是 2.5Gbaud——须按目标速率重取器件手册单点，或用 Giggenbach 250 光子/bit 换算（本轮未取到 2.5Gbps 直检的权威单点）。
2. **相干 F 取值**：理想地板 3dB(F=2) 确证，EDFA 前放 NF≈5dB(F≈3.16) 为典型但依器件变化（4.5–6dB）——具体终端须厂商数据或设计假设。
3. **提前量是否主动补偿**：θ_PA（LEO 几十 µrad）与发散角同量级——终端做不做 point-ahead 主动补偿，决定它是"确定性指向损耗"还是"已消除"，影响余量。
4. **指向损耗模型三选一**：Ruze-like（现用）/ 高斯指数（arXiv）/ β-分布均值（DLR）——小角一致、大角发散，须定一个标准并统一 σ 定义。
5. **太阳规避并入方式**：需真实日月星历算对日矢量，本仓库几何是否已具备待确认；并入"完整可用度"须避免与建链/抖动重复计中断。
6. **JLT 32(7):1295 逐符号灵敏度闭式在付费全文内**：本轮只取到摘要级；若要收敛相干灵敏度到逐符号式，需付费全文。

## 附录 B：当前实现已确认问题清单（按严重度，供追溯）

**Critical**：① 调制↔灵敏度脱钩（选 OOK 仍算 PPB=20，虚高 13–20dB）。

**Major**：② 发射增益缺发散角/效率口径统一；③ 两套互斥噪声模型并存（余量用 PPB=20、C/N₀ 用 N₀=hν）；④ 体制错配（CONDOR 直检光学套相干 100G 灵敏度）；⑤ 立论错误（注释否认载波）；⑥ N₀=hν 忽略 EDFA NF/ASE（低估 3–6dB）；⑦ 提前量角 θ_PA 完全未算；⑧ worst-case 取最大 range 单点致多普勒近零；⑨ "完整可用度"漏太阳/建链；⑩ designMargin 不 gate；⑪ 指向 θ_e 光束定义错（低估 4×）；⑫ 符号率/带宽/线宽缺；⑬ targetBer/FEC 对 PPB 无影响。

**Minor**：⑭ 抖动 σ 轻微双重计入（默认仅 0.12dB）；⑮ 太阳仅静态参数；⑯ 跟踪转率未算；⑰ 引擎兜底默认与 UI 不一致；⑱ 多普勒注释 10× 错（6.45→0.645）；⑲ 净捕获比注释漏 π²/16；⑳ 偏置按静态 dB 未进 Rician 衰落；㉑ 参考算例 +1.2dB < designMargin 3dB。
