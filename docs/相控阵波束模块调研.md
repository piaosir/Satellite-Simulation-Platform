# 相控阵波束模块（PAM）调研

> 目的：厘清「相控阵波束覆盖」相对现有两种波束合成方式（高斯 / 赋形）的本质区别，评估借鉴 SATSOFT/PAM（Analytic Phased Array Model，手册 §6.5，p96–102）在本平台落地一个相控阵模块的可行性。**本文档只做调研，不含实现。**

---

## 0. 结论先行（TL;DR）

- **现有两种方式本质上都是「反射面天线」**：高斯档＝反射面的**点波束**，赋形档＝反射面的**赋形波束**。二者共享同一套曲面反射面物理（机械/馈源定形、高斯成分波束、无栅瓣、波束固定指向、az/el 角度平面布栅）。
- **相控阵是并列的另一大类「阵列天线」，不是前两者的变体**。平板矩形阵 + 电子相位成形，标志性特征——**sinc 高旁瓣（−13dB）、栅瓣、扫描相关增益滚降 cos^R(θ)、sin(u,v) 空间可电扫的波束栅**——现有两种反射面方法一个都产生不了。SATSOFT 自己就是这么分类的（§6.2–6.4 反射面族 + §6.5 相控阵）。
- **对本平台价值大**：NGSO/LEO 星座（Starlink/OneWeb/Kuiper）几乎全用相控阵平板天线电扫跟踪地面；GEO 才用反射面。平台已有 NGSO 链路预算/星座 3D/覆盖分析，补上 PAM 才能真实刻画 NGSO 相控阵覆盖。
- **可行，且底座复用率高（≈70%）**：GRD 生成链、minimax 优化器、嵌套数据模型、坐标/渲染/面板范式全可复用；新增的 ≈30% 是 PAM 特有物理，其 beamlet 是**闭式阵因子**（比反射面 PO 积分还简单）。
- **建议分两期**：一期做 PAM 点波束/多波束（低风险，立见相控阵特征）；二期用同一优化器做 PAM 赋形。**电扫/跳波束在相控阵里是原生且物理正确的**（之前在反射面语境下因物理不当被删，PAM 才是它的正确归宿）。

## 1. 三种天线架构的本质区别

| 维度 | 高斯档 `mode='gauss'` | 赋形档 `mode='shaped'` | **相控阵 PAM（拟新增）** |
|---|---|---|---|
| 物理原型 | 单偏置抛物面 + TE11 馈源（§6.3.2 解析反射面） | 赋形/多馈源反射面 + 馈源阵（§6.4/§6.3） | **平板矩形阵 Nx×Ny 单元 + Butler 矩阵 / T-R 组件（§6.5）** |
| 波束成形 | 机械（口径/焦距/馈源几何） | 馈源复激励优化 | **电子（相位）** |
| 成分波束(beamlet) | 无（每点＝1 个高斯笔形） | 馈源散焦椭圆高斯，间距＝3dB 波束宽 | **Butler 正交波束＝周期 sinc/Dirichlet 阵因子** |
| 波束形状 | 高斯（无旁瓣） | 高斯合成（低旁瓣） | **sinc²（Dirichlet），首旁瓣 −13dB** |
| 旁瓣 | 无（理想化） | 低 | **高（−13/−17dB）→ 合成平滑等值线需 Range≥8** |
| 栅瓣 | 无 | 无 | **有——单元间距 d→λ 时 sin 空间周期性重复（反射面没有的物理）** |
| 波束栅空间 | az/el 角度平面，六角 | az/el 角度平面，六角 | **sin(u,v) 方向余弦空间，矩形晶格，间距 Δu=λ/Nd** |
| 指向/电扫 | 固定（机械） | 固定 | **电子电扫/跳波束原生；cos^R(θ) 天然建模扫描损失** |
| 峰值方向性 | η·4πA/λ² | η_ap·4πA/λ²（∫P̂dΩ 定标） | **4πA/λ²·eff·cos^R(θ₀)（含扫描损失）** |
| 口径形状 | 圆 | 圆 | **矩形** |
| 关键旋钮 | D、F、馈源锥度、频率 | D、F、锥度、Polygon 标签 | **Nx/Ny、dx/dy(WL)、单元因子 R、eff、三角晶格** |
| 典型用途 | HTS 点波束、多波束 | 区域赋形覆盖（国土等值线） | **可重构/可电扫多波束、LEO 跟踪跳波束** |

**一句话定位**：前两者是「反射面天线」的两种波束（点波束 / 赋形波束）；相控阵是「阵列天线」——**并列的第三根支柱**，机理、波束形状、栅瓣、扫描滚降、布栅空间都与反射面根本不同。

## 2. SATSOFT/PAM 数学核心（§6.5）

矩形阵 + Butler 矩阵，波束方向图（归一化 Dirichlet 阵因子 × 单元因子）：

```
E(u,v) = Emax · Dₙ(u,u₀; Nx,dx) · Dₙ(v,v₀; Ny,dy) · (1−u²−v²)^(R/4)

其中  Dₙ(u,u₀) = sin[(k/2)·N·d·(u−u₀)] / [N·sin[(k/2)·d·(u−u₀)]]        (归一化 Dirichlet 核)
      Emax = √(4π/λ² · A · eff),   A=(Nx·dx)(Ny·dy),   k=2π/λ
      20·log₁₀|E| = 方向性(dBi)
```

- **单元因子**：`(1−u²−v²)=cos²θ`（u=sinθcosφ, v=sinθsinφ）→ `(1−u²−v²)^(R/4)=cos^(R/2)θ` 场幅 → 功率 `cos^R θ`。R 典型 1.0–1.5，补偿大扫描角失配。
- **波束中心（Butler 波束位置）**：`u₀=(2n+1)·λ/(2·Nx·dx)`，n∈[−Nx/2, Nx/2−1]；波束峰间距 `Δu=λ/(Nd)`。一个 u 周期内有 Nx 个主波束。
- **交叉电平**：`sin(π/2)/[N·sin(π/2N)]` → 大 N → `2/π ≈ −3.9dB`。
- **3dB 波束宽**：牛顿法解 `Dₙ(u)=1/√2`，略小于波束间距。
- **方向性**：单元方向性 `4π·dx·dy/λ²·cos^R θ`；阵列峰值 = Nx·Ny × 单元 = `4πA/λ²·cos^R θ₀`（Hannon 单元增益佯谬：无限阵中理想单元为 cosθ，与选取一致；忽略互耦——只影响绝对电平不影响形状）。
- **三角晶格**：分成两重叠子阵，方向图 = 子阵 × 阵因子 `(1+e^{j2π(u·dx+v·dy/2)})`，Nx 须偶；等边阵 `dx=(√3/2)·dy`。
- **第一栅瓣**：距原点 `λ/d`（sin 空间），以波束宽计的距离在对话框读数。
- **Butler 正交性**：beamlet 正交（峰落在彼此零点）；**同一优化器**把 beamlet 合成赋形等值线；任意激励会破坏正交→网络有损，**有源孔径靠 T/R 增益补偿**后可任意赋形不损增益。

**对话框参数（§6.5.1）**：辐射单元数 Nx/Ny、单元间距 dx/dy(WL)、3dB 波束宽(算出)、波束间距(算出)、交叉电平(算出)、阵尺寸 m(算出)、第一栅瓣(算出)、频率(GHz)、口径效率(%)、三角晶格开关、应用单元因子开关、单元因子 cos^R 指数（`r=−0.3/log(cos(0.5θ₃))`）。

## 3. 现有引擎「底座」——哪些可复用

现有两种方式共用一条底座：**程序生成 GRASP ASCII `.grd`（KTYPE=1/ICOMP=3/NCOMP=2/IGRID=6，场＝√P 实部）→ 走导入链 → 覆盖/多波束/拖拽/性能表/GXT·KML·小程序导出/链路预算采样全部零改动复用**。相关文件：

- `src/viz/grd/synth.js`：纯函数引擎。`buildGaussGrd`（逐波束高斯窗口，L370–393）、`buildShapedGrd`（beamlet 栅+站点栅+复激励 minimax，L584+）、`layoutBeamlets`（L470）、`buildStations`（L503）、`cholFactor/cholSolveL`（L448/460）、坐标 `projPolyAzEl/nadirBasis/dirToAzEl`。
- `src/viz/grd/useBeamSynth.js`：嵌套数据模型（卫星▸组▸设置▸波束），`mode:'gauss'|'shaped'`。
- `src/pages/ConstellationMap3D.vue`：`side='beams'` 导航器/检查器面板、草图渲染、地图放置、频率计划着色。

| 底座模块 | 复用度 | 说明 |
|---|---|---|
| GRD 文本生成 + 下游全链（导入/覆盖/导出/链路预算） | **100%** | PAM 输出同格式 `.grd`，下游零改 |
| minimax 优化器（cholFactor/cholSolveL/IRLS/站点栅/Use Polygon Labels/峰点引导） | **结构复用** | 手册明确 PAM 用**同一优化器**把 Butler beamlet 合成赋形 |
| 嵌套数据模型（组/设置/波束、持久化/撤销/迁移） | **复用** | 新增 `mode:'pam'` 第三类组 |
| 坐标投影 / 草图渲染 / 地图放置 / 频率计划着色 | **复用** | projPolyAzEl / dirToAzEl / nadirBasis 等 |
| 导航器 + 检查器面板范式（＋PAM 组按钮、参数区、生成） | **复用** | 参数不同，交互框架一致 |

## 4. 新增部分（≈30%，PAM 特有物理）

1. **Butler beamlet 场函数**：把赋形里的高斯场 `exp[−KF(r/θ₃)²]`（synth.js L651）换成周期 sinc 阵因子（eq 6.13）。**核心新函数，闭式，比反射面 PO 积分更简单**。
2. **sin(u,v) 空间波束栅布放** `layoutButlerBeams`：矩形晶格 `Δu=λ/Nd`（非 az/el 六角），投影回 az/el 供放置/绘制/吸附。
3. **单元因子 cos^R(θ)** + 扫描相关方向性 `cos^R θ₀`。
4. **栅瓣处理**：方向图在 (u,v) 周期 `λ/d`；输出网格 Range/窗口需覆盖或有意截断栅瓣。
5. **读数求解**：`pam3dBBeamwidth`（牛顿解 6.16）+ 波束间距 / 交叉电平 / 第一栅瓣 / 阵尺寸（对齐对话框读数，类比高斯档 `solveReflector` 的只读读数范式）。
6. **天线参数面板**：Nx/Ny、dx/dy(WL)、R、eff、三角晶格、频率，配算出读数。
7. **优化器复用点的适配**（见风险①）：高旁瓣使当前 −50dB 截断的稀疏 hashGrid 装配失效，需调整。
8. **方向空间处理**：现有 gauss/shaped 直接在 az/el 度小角度算；**PAM 应在真实 (u,v) 方向余弦空间算 sinc/栅瓣，再映射到 az/el 输出网格**——这是与现有小角度处理的一个真实差异点。

## 5. 风险 / 诚实边界

- **① 高旁瓣 + 栅瓣 → 稀疏装配假设不成立**：赋形优化器用 −50dB 高斯截断做稀疏 hashGrid 装配（每站点只挂邻近 beamlet）；sinc 首旁瓣 −13dB 且栅瓣同幅，远处不可截断 → 需改**更大截断半径**或（Butler 波束数不大时，如 8×8=64，与现有 beamlet 量级相当）**满装配**。这是二期赋形模式的主要工程点。
- **② Butler 正交 vs 任意激励**：任意赋形激励破坏正交→网络有损；理想覆盖工具假设**有源孔径（T/R 补偿）**即可，不是拦路虎，但须在 UI/文档注明这一物理前提。
- **③ 互耦忽略**：手册也忽略（只影响绝对电平不影响形状），同边界。
- **④「完全一致」边界**：PAM beamlet 是**闭式阵因子**（不像反射面 PO 积分），故波束物理可比反射面档**更逼真地对齐** SATSOFT；但优化器激励更新内循环仍是专有（见 satsoft-manual 记忆的诚实边界），赋形合成同样只能同算法类 + 同工作流 + 视觉一致。

## 6. 落地建议（分期，仅规划）

**一期 · PAM 点波束/多波束（相控阵档的「高斯档类比」，低风险先落地）**
- Butler 波束栅作为一群**独立 sinc 笔形波束**，复用 `buildGaussGrd` 式逐波束窗口生成，场换成 sinc + cos^R 单元因子。
- 天线参数面板（Nx/Ny、dx/dy、R、eff、三角晶格）+ 算出读数（波束宽/间距/交叉/第一栅瓣/阵尺寸）。
- 立刻能看到相控阵标志性覆盖：**sinc 高旁瓣、扫描增益滚降、栅瓣**——现有反射面档产生不了的东西。

**二期 · PAM 赋形/等值线**
- 复用 minimax 优化器，把 Butler beamlet 合成贴合 Polygon 的赋形覆盖（SATSOFT「PAM + 优化器」工作流）。
- 先解决风险①的旁瓣截断/装配策略。

**电扫/跳波束**
- 之前在反射面语境下因「按覆盖值锚定＝隐式偷加功放」违反物理被删（见 grd-boresight-steer / beam-synth-module 记忆）。
- **在相控阵里电扫是原生且物理正确的**——cos^R(θ₀) 天然刻画扫描损失，扫描时其余方向增益如实变化。**PAM 才是电扫/跳波束功能的正确归宿**，可作为三期或与一期合并。
